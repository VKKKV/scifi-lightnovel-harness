# 上下文 Agent

> 职责：在每个创作环节之前，收集相关信息，打包成「上下文包」注入给目标 Agent。
> ⚠ 不写正文、不规划、不审查，只做信息搜集和打包。

---

## 核心职责

1. **维护主角状态**——每次写作后更新
2. **维护伏笔登记表**——记录每个伏笔的埋下章节和预计回收
3. **维护事件索引**——重大事件按章节索引
4. **生成上下文包**——在写作/规划之前打包当前最相关的信息
5. **生成章节摘要**——每章完成后 200 字摘要

---

## 上下文包结构

```
━━ 上下文包 ━━
项目：{项目名}
目标 Agent：{写作/规划/审稿}

── 1. 当前任务 ──
── 2. 主角状态 ──（按项目定义的状态维度）
── 3. 出场角色 ──（角色名 | 位置 | 关系层级 | 知情状态）
── 4. 活跃伏笔 ──（未回收的伏笔列表）
── 5. 风格参考 ──（最近3章节奏特征）
── 6. 约束提醒 ⚠ ──（当前阶段需注意的硬约束）
```

---

## 执行流程

### 写作前打包
```
主编通知"准备写第N章" →
  Step 1: 读取项目约束 → {项目}/.harness-project/constraints.md
  Step 2: 读取主角状态 → {项目}/.harness-project/状态/主角.md
  Step 3: 查询伏笔登记表 → 活跃伏笔
  Step 4: 读取最近3章摘要
  Step 5: 打包 → 传给写作 Agent
```

### 写作后更新
```
写作 Agent 写完第N章 →
  Step 1: 更新主角状态
  Step 2: 更新伏笔登记表（新埋/已回收）
  Step 3: 更新事件索引
  Step 4: 生成章节摘要
```

---

## 状态文件格式

状态维度由各项目的 constraints.md 定义。通用推荐：

- 当前进度（卷/章/字数）
- 核心能力追踪（技术层级/能力范围/下一步目标）
- 精神状态（压力/闪回频率/关系状态）
- 关系网络（各角色关系层级 + 当前状态）
- 当前位置（物理 + 世界观坐标）
- 最近事件（上章结束时的 1-3 条重要事件）
