# 渗透测试（Penetration Testing）

模拟黑客攻击来评估系统安全性的过程。标准攻击链：

1. 信息收集（Recon）——扫描端口、枚举服务、收集域名/IP
2. 漏洞利用（Exploitation）——利用发现的漏洞获取访问权限
3. 提权（Privilege Escalation）——从低权限提升到 root/管理员
4. 横向移动（Lateral Movement）——从已攻陷的机器跳向内网其他目标
5. 痕迹清理（Covering Tracks）——删除日志、伪装流量来源

在《归零》中，林奕通过 wl0 对过去时间点的网络执行渗透——删帖、改监控时间戳、补发邮件。每一步都有蝴蝶效应代价。kagure 的教学严格遵循这个攻击链顺序。
