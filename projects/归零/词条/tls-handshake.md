# TLS/SSL 握手

HTTPS 的加密基础。客户端和服务器在传输数据前协商加密方式：

1. ClientHello：客户端发送支持的加密套件列表 + 随机数
2. ServerHello：服务器选择加密套件 + 发回证书 + 随机数
3. 证书验证：客户端验证服务器证书是否可信
4. 密钥交换：双方用随机数生成对称加密密钥
5. 加密通信开始

关键字段：SNI（Server Name Indication）——TLS 握手包中明文传输的目标域名。即使流量加密，SNI 仍然泄露了用户在访问哪个网站。这是叶岚在 CTF 赛题中指出林奕「filter 写错了」的切入点——流量分析时 SNI 是最常见的信息泄露点。
