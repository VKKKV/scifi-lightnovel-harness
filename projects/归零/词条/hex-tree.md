# hex tree

浅羽谦治服务器上的隐藏目录结构。文件名和目录名全是 16 进制乱码。只有终端持有者（林奕）能在 P53 上通过 `.meta` 索引文件解码阅读。

hex tree 是浅羽美佐留下的——她把终端相关知识编码进了谦治的服务器。谦治知道它的存在但从没碰过。

第一层映射通过 `.meta` 文件提供（键值对：hex→英文标签）：
- `0x7a` → gateway
- `0x3f1a` → observer
- `0xa02` → threshold
- `0xffff` → merge
- `0x01` → Ryou

更深层文件有嵌套加密。部分在 P53 上查看时出现系统级注释——不是 `.meta` 提供的，是终端系统的行为。

千夏对密码学的兴趣来自 hex tree——她发现了加密文件但打不开，开始自学解密。
