# 日志清理（Log Cleaning）

渗透攻击的最后一步——覆盖入侵痕迹。标准清理目标：

- shell history（`.bash_history`、`.zsh_history`）
- 系统日志（`/var/log/auth.log`、`/var/log/syslog`、`/var/log/wtmp`）
- 应用日志（Web 服务器 access log、数据库查询日志）
- 命令执行时间戳（`history -c && rm ~/.bash_history`）

在《归零》中，kagure 教林奕的第一批技术就包括痕迹清理。不是因为这是最难的——而是因为这是最容易忽略的。「你渗透完不清理日志——等于在犯罪现场留了指纹。」

林奕的日志清理能力随着教学深入逐步提升：从只清自己的 → 清整个时间段的 → 伪造假日志覆盖真日志。
