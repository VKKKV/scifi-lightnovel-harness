## Part 4 剩余：关系深化 + 能力成长

### 高柳 B1 谈话

林奕按约去 B1。不是演出日——七点前锁着门，高柳有钥匙。地下室只有两个人。

高柳开门见山：「你帖子上描述的网卡现象——我研究了六年。」

他的研究方向是量子信息科学。不是终端——是「退相干历史的宏观可观测效应」。他没有终端。但他从学术角度推导出了一个理论框架：如果世界线分支是真实的，那么分支之间的信息残留应该可以通过量子态坍缩的偏差来观测。

「你看到的那些——不是超自然。是物理。是我们还没建好观测工具的物理。」

他不问林奕的终端是什么、怎么运作。他只问观测数据：git log 的格式、commit 的密度、anomaly 标记的模式。林奕给他看了 daemon.md 的规则。高柳看完了，没评价。问了一个问题：

「你有没有试过 `git branch`。」

林奕没有。他不知道 git branch 在这个语境下意味着什么。

高柳没解释。只说：「别试。先搞清楚 `git log` 能告诉你什么。分支是后面的事。」

离开时高柳说了最后一句话：「你的 kagure——如果他是我说的那个人——他不会告诉你全部。因为他自己也不知道全部。他只是比你早六年撞上这堵墙。」

**作用**：建立科学解释框架（量子信息+退相干）。为后续「教授从物理学角度解释世界线」铺路。高柳不是终端持有者——他是从学术角度推导的人。

### 叶岚正式登场

林奕参加一个本地 CTF 线下赛（DEF CON CTF 预选，东京站）。不是为了赢——是 kagure 说的「你该和真人打了」。

一道流量分析题。林奕在 Wireshark 上卡了一小时——pcap 包里全是加密流量，他找不到切入点。

旁边一个女生路过。看了一眼他的屏幕。

「你 filter 写错了。」

叶岚。帆布包上有 HITCON、SECCON、DEF CON CTF pin。和便利店门口那个连帽衫女生是同一个人。但这次她没走——她坐下来，用三分钟指出了流量分析的正确切入点（TLS 握手的 ClientHello 里有 SNI 泄露）。

林奕解出那道题。叶岚解了全场最高分。

赛后高柳在 B1 把叶岚介绍给林奕：「你们应该认识了。」

叶岚看了林奕一眼。「便利店那个 filter。」

林奕点了一下头。

高柳没有多解释。只说：「你们两个——一个是渗透，一个是分析。互补。」

**作用**：叶岚正式进入主角团。互补关系确立（暴力渗透 vs 学术分析）。CTF 场景自然建立她和林奕的技术等级差——她比他强，但方向不同。

### kiri 首次出现

林奕翻 git log 时发现了一个他之前没注意到的模式：有些 commit 不是他操作产生的。commit hash 极短（4 位），commit message 是数字（1、2、3...），时间戳比他任何一次渗透都早——有些甚至早到 2019 年之前。

他把这些 commit 筛出来。分布在整个世界线树上——不是某一条分支，是所有分支都有。

commit author：kiri。

不是他的操作。不是 kagure 的。不是任何一个他认识的人。

是另一个实体——或者另一个曾经使用过这个终端系统的存在。

daemon.md 新规则：「git log 里有不属于我的 commit。作者是 kiri。不知道是什么。分布在整个世界线树上。有些比 2019 年更早。」

**作用**：埋下 kiri 伏笔。林奕意识到「这个终端不是他一个人的」。为卷二-三追踪线铺路。

### 千夏感情线 + 闪回加剧

日常场景推进：
- 千夏在柜台后面写代码，林奕在旁边看 git log。两个人各干各的，偶尔说一句话。不是教学——是待在一起。
- 千夏问林奕「你最近是不是没睡好」。不是关心的语气——是陈述事实。
- 赵逸风注意到林奕在杂货店的时间比在公寓多。

闪回加剧：
- 第二次闪回：便利店荧光灯的光。不是现在的光——是另一种光。更白。更冷。千夏不在柜台后面。不知道为什么他知道这一点。
- 第三次闪回：一个声音。键盘声。不是林奕的键盘——是更旧的、机械轴的声音。PS/2 接口的那种。服务器房间的声音。
- 林奕开始在闪回后头疼。不是剧痛——是那种用眼过度的酸胀。daemon.md 记录了频率。三次。间隔在缩短。

**作用**：感情线自然推进（不是告白/心动——是待在一起的舒适感）。闪回频率增加暗示精神压力。为后续「蝴蝶效应危机」铺情绪基础。

## Part 5：团队运作 + 异常加剧

### 三角技术线

林奕+叶岚+kagure 形成三条线：

- kagure（线上）：教渗透技术。只给题不给讲解。Ch10-14 密集教学后进入沉默期——他的沉默本身就是信息。Ch24 长警告是沉默后的爆发。
- 叶岚：用学术方法分析。她不碰终端——但她在网络层做流量分析，用纯技术手段追踪异常。她追踪到林奕操作产生的流量残留——这些残留的特征不符合任何已知网络协议。（「量子退相干宏观签名」→ 移至卷二，待高柳框架展开后由叶岚说出）
- 林奕：执行者。kagure 教技术，叶岚给方向，他动手。

三个人各自知道一部分真相，但没有完全对齐。kagure 知道系统但不共享。叶岚不知道系统但能从数据推导。林奕两边都不完全信任——但他需要两边。

### 服务器深度探索

千夏给了林奕和叶岚完全的服务器访问权限。叶岚用 her own tools 扫描服务器——发现 hex tree 不只是目录结构，每个 hex 文件内部还有嵌套的加密层。

`.meta` 文件（林奕之前发现的）只有第一层映射。更深层的 hex 文件需要密钥。

林奕把整个 hex tree 拷到 P53 上。叶岚用 her tools 分析深层文件——发现每个 hex 文件内部有嵌套的加密层。.meta 只有第一层映射。更深层需要密钥。

叶岚在分析过程中发现了一些东西——不是系统自动翻译，是她在 hex 文件的二进制结构中找到了内嵌的文本片段。像是美佐或者团队成员在文件创建时写入的注释——被编码在文件头的元数据里，不是明文，但用正确的工具可以提取：

「Observer threshold crossed at 1919810.」
「Cascade initiated. Subject: Asaba Chinatsu.」
「Branch pruned. Residual: negative.」

林奕看不懂全部。但他看懂了两个词：千夏的名字，和「branch pruned」。

daemon.md：「服务器里有千夏的名字。和世界线编号放在一起。她被某个系统标记了。」

**作用**：服务器不只是信息存储——它记录了系统对千夏的操作。为后续「必死节点」铺路。同时暗示终端系统比林奕想的更复杂——不只是观测面板。

### 必死节点首次观测

林奕在一次渗透操作中——他在 git log 里看到了一个他没见过的标记：`death_node`。

不是一次——是多次。不同的世界线分支上，同一个名字反复出现：浅羽千夏。不同的死法。不同的时间。但每一次——都在某个节点上终止。

七种方式。七条分支。七次死亡。

在每一条他没有干涉过的世界线上——千夏是必死节点。

林奕盯着 git log。屏幕的光照在他脸上。他的手没有抖——但他把 daemon.md 里所有关于千夏的记录都删了。不是因为不重要。是因为太重要了。重要到他不敢让这些记录存在在任何地方——即使是他的笔记。

daemon.md 最后一行：「我不知道为什么。但我不会再让她死。」

**作用**：必死节点首次观测。情感转折——从「测试系统」变成「保护千夏」。为后续蝴蝶效应危机铺路。

## Part 6：蝴蝶效应危机

### 连锁反应积累

林奕的渗透操作已经积累了数十次。每一次都像往水里扔石头——涟漪向外扩散，互相叠加。

最初的异常是细小的：
- 新闻里出现了一个他不认识的名字——但这个名字的缩写和他修改过的一条记录有关。
- 赵逸风某天说了一句「你最近是不是——」然后停住了。他自己也不知道自己要说什么。那句话的逻辑链从他的嘴里出来的时候就断了。
- 千夏在服务器上看到了一些她不理解的流量——不是攻击，是某种模式。她在本子上记下了。

然后越来越大：
- 一个他渗透过的服务器——管理员因为日志异常报了警。警察开始调查。
- 一个他改过交通信号的路口——发生了另一起事故。不是千夏。是一个陌生人。林奕在新闻里看到了。
- 他的 git log 里出现了一条异常 commit——cascade 数据的密度和频率超出了他之前观察到的所有模式。他自己推导出结论：蝴蝶效应在加速。不是系统告诉他的——是他从数据里看出来的。

**作用**：蝴蝶效应从「可忽略」升级到「不可忽略」。林奕开始意识到代价的规模。

---

### 千夏被波及

千夏在服务器笔记里记录了异常：「最近一个月，服务器流量有三次不是我触发的。来源不明。协议不是常见的——像是某种低层的握手。」

她不理解这些流量。但她把它们记在本子上——和她记指示灯颜色的方式一样。她不知道这些流量是林奕的渗透操作在世界线上的残留。

林奕看到了她的笔记。他没有告诉她真相。

**作用**：千夏在不知情的情况下被卷入。林奕的隐瞒加重了他的精神负担。

---

### kagure 的警告

kagure 在私信里发了一段长消息——比以往任何时候都长：

「你改得太多了。因果律开始重新计算——不只是你改的那个点。是从那个点往后的全部。你身边的人——你控制不了。」

「你救的那个人——每一次你救她，世界线就在她身上多标记一次。你让她活着——但你让她的存在变得越来越不稳定。系统在纠正。你在抵抗纠正。结果是——裂缝。」

「别再渗透了。至少别再渗透和她相关的任何东西。」

林奕没有回复。他知道 kagure 说得对。但他做不到。

**作用**：kagure 给出明确警告。林奕面临选择——继续渗透（保护千夏）还是停止（保护所有人）。这是卷一的核心矛盾。

### 赵逸风事件

某个渗透的连锁反应击中了赵逸风。

不是物理伤害——是信息层面的。赵逸风收到了一封他不认识的邮件——来自一个他不认识的发件人。邮件内容是一段加密文本。他看不懂。但他本能地知道这是给林奕的——因为发件人的名字里有「wl0」。

赵逸风没有问林奕。他把邮件转发给了林奕——没加任何说明。

林奕看到邮件的时候，凌晨三点。赵逸风在客厅吃泡面。两个人隔着一道墙。谁都没有敲对方的门。

凌晨三点十五分。赵逸风走到林奕桌前，放了一碗泡面。没说话。回去了。

林奕看着那碗泡面。他知道赵逸风不懂计算机。不懂终端。不懂蝴蝶效应。但赵逸风懂人。他知道林奕在承受什么——即使他不知道是什么。

**作用**：赵逸风从「不知道发生了什么」变成「知道林奕在承受但不知道是什么」。凌晨三点泡面成为两人关系的标志性场景。

### 求助（转折点）

林奕做了他从未做过的事——求助。

他把千夏、叶岚、高柳聚在 B1 地下室。不是演出日——七点前锁着门。四个人坐在舞台上。林奕把 P53 打开。

他没有解释终端的机制——他自己也不完全理解。他说的是事实：
- 他有一台能连接过去网络的电脑。
- 他用它改了一些事情。
- 每次改动都有连锁反应。
- 千夏在某个他不理解的机制里被标记了。
- 他控制不了了。

千夏第一次知道父亲留下的服务器是什么。她没有哭。没有震惊。她坐在那里，手放在膝盖上，听着林奕说完。然后她说了一句话：「所以服务器里那些流量——是你的。」

叶岚的第一反应不是恐惧——是「有意思」。她看了接口状态，做了一个判断（不在地面网络栈），说「我需要想想」。没有列问题清单——她需要时间消化。（三问题结构→改为更自然的单点深入）

高柳沉默了很久。然后开始提问题——不是「你为什么现在才告诉我们」，是「death_node 标记——你能看到触发条件吗」。

赵逸风不在。林奕没有叫他。这是他唯一一次后悔没有叫赵逸风——因为赵逸风是唯一会说「所以你现在这么累——是因为这个」的人。

**作用**：团队知情时间线同步。林奕从独行者变成团队核心。每个人的第一反应都符合性格。千夏的服务器秘密和终端秘密在这一刻汇合。

### 卷尾钩子

千夏回到便利店。上二楼看服务器。在 hex tree 的最深处——她发现了一个新的目录。不是之前就有的——是最近才出现的。目录名不是 hex——是明文：

`passwd.vault`

她打不开。文件头是一串她看不懂的编码。但她知道这不是系统自动生成的——是有人写进去的。或者——是某个系统在某个时刻留下的。

她把文件头抄在了柜台后面的本子上。

她不知道这是密码学的开始。
