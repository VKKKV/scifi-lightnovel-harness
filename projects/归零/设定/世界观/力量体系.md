# 力量体系：跨时空 APT 攻击

## 核心概念

### 终端的起源：超级量子计算机

几十年前，幕后组织资助了一群全球顶尖的量子物理、计算机科学、密码学专家，建造了一台**超级量子计算机**。这台机器拥有超级 AI——能够推演过去未来、干涉因果律。

科学家团队通过一个终端操作这台计算机。终端呈现为类 Unix 风格——这是科学家们设计的用户界面，翻译成人类能理解的形式。

**爆炸**：实验失控。团队中一群良心发现的科学家（千夏母亲美佐是其中之一）人为制造了一场爆炸，试图摧毁量子计算机和终端。

**不确定态**：爆炸后，量子计算机的命运不明。终端变成了一种不确定的状态——一个或许连接又或许不连接到那台量子计算机的接口。wl0 虚拟网卡是这个接口在普通电脑上的映射。

林奕不知道他连接的是什么——是一台还存在的量子计算机？还是量子计算机的残余投影？还是某种他自己也无法理解的东西？他只知道——它能用。

### wl0：时空 VPN 隧道

虚拟网卡 `wl0` 不是「世界线操作面板」——它是一个**能连接并代理到过去时间点计算机网络的时空 VPN 隧道**。

当林奕在终端上输入 `ssh root@worldline.1919810` 时，他连接的不是「宇宙系统」——而是**1919810 号世界线分支上、某个时间点的网络环境**。终端呈现为类 Unix 风格——这只是一个用户界面，不是系统本质。

**IP 地址中的时间坐标**：
- `192.168.0.1.114514`——最后一串数字（114514）编码了时间偏移量
- 林奕最初只是偶然发现了这个地址
- 后期他学会构造自己的目标 IP，精确连接到过去某个时刻的某个网络设备

### git 命令：观测面板

- `git log --oneline`：查阅当前世界线的「事件快照」——谁做了什么、什么时间、什么结果。**只读**。不能修改现实
- `git reset`：将观测焦点回退到过去的某个 commit 节点——从那个节点重新观测。**不改变现实**——改变的是「林奕从哪个时间点出发」
- `git branch`：标记不同的观测路径——**不能创造新世界**，只是为观测路径打了标签

**关键词**：git 是后视镜和地图——不是方向盘。

## 真正的操作方式：跨时空渗透

林奕改变过去的唯一方式，是通过 wl0 隧道连接目标时间点的网络设备，利用黑客技术进行**渗透**。

### 操作流程

```
1. 定位目标：
   git log 发现未来/过去的事件 → 确定目标 IP（含时间坐标）
   
2. 建立隧道：
   ssh root@worldline.{编号} → 隧道连接 → 到达目标时间点的网络环境

3. 渗透攻击：
   在该时间点的网络上进行 CTF 式的渗透攻击
   → 拿到目标设备的 shell
   → 修改数据/配置/记录

4. 现实更新：
   渗透成功 → 目标时间点的数据被改写
   → 从那个时间点往后的历史重新计算 → 现实世界即时更新
```

### 能力分层（= CTF 技术分层）

| 层级 | 渗透目标 | 改变范围 | 所需技术 | 示例 |
|------|---------|---------|---------|------|
| **初级**（Web/数据库） | 旧 BBS 帖子、网站 CMS、公开数据库 | 改变信息流向、删除某个页面/帖子 | SQLi、XSS、文件包含 | 删除某人发布的帖子 → 某条信息从未传播 |
| **中级**（内网/特定设施） | 医院系统、交通信号、学校教务、小型 ISP | 改变个人的生死、事故、人生轨迹 | 内网渗透、协议攻击、提权 | 修改三年前医院电子病历 → 某人治疗方案不同 → 存活 |
| **高级**（核心基础设施） | 电网、金融网络、基础通信设施、世界线系统底层 | 改变大规模事件、甚至触及系统本身 | 顶级 Pwn、0-day 挖掘、内核漏洞 | 修改电网调度记录 → 某场停电没发生 |

### 核心优势

CTF 技术不是背景板——是林奕的**实打实的战斗技能**。他每学会一道 Pwn 题、每理解一种加密模式的弱点、每掌握一种提权方法——都是在扩展他能改变的现实范围。不再是「系统赠与能力」——是他自己在成长。

**一个导论课水平的 noob** 最初只能改掉某个旧 BBS 上的一条帖子（他花了一整夜才找到一个 SQLi 注入点）。随着他变强，他能改变的东西越来越大。他学的每一行代码、每一道 CTF 题、每一个 kagure 教的渗透技巧——都直接转化为他能拯救的人。

## 代价体系：蝴蝶效应

### 每次渗透都是蝴蝶翅膀

每次渗透在目标时间点的网络上修改数据——被修改的不止是目标。历史从修改点开始重新计算，连锁反应超出林奕的预测范围。他改了一个人的死亡——他不知道那条世界线上还有谁因此活着、谁因此没活。

### 代价体现

- 渗透规模越大 → 蝴蝶效应范围越广 → 林奕周围的人被波及
- 他不知道这次修改会波及谁——直到他看见结果
- 不是系统惩罚他——是因果律本身没有安全边界

### 世界不和你说话

系统不会给林奕发消息。林奕知道的所有规则——都是他自己推导的。

他能看到的东西：
- `git log` 中的自动 commit（事件记录）——**只读**
- 他自己的操作在现实中引发的连锁反应

他看不到的东西：
- 没有任何「系统」会告诉他「你做对了」「你做错了」「其他观测者在看你」
- 没有导师，没有提示
- 一切因果关系——他自己推。从结果反推原因，从现实反推规则

### 唯一持有者

同一时刻只能有一人持有终端权限。目前：**只有林奕**。历史上无前任持有者——林奕是第一个。终端通过 bootstrap 循环选中了他，在他之前没有活跃的观测者。

终端以一种不确定的形式存在——选中林奕的原因不解释。这是全书唯一一抹奇幻色彩。

## 升级机制

林奕的「升级」靠两点：
1. **技术成长**：CTF + kagure 教学 + 实战渗透经验 = 能攻破更多系统
2. **记忆保留**：多个分支上的技能通过闪回交汇 → 在一条世界线上积累的经验不下线

**不是系统给能力——是他自己练出来的。**

## 回溯机制（写作工具）

蝴蝶效应积累到临界点时，因果链会自然回溯到某个时间点——像橡皮筋拉到极限弹回去。

规则：
- 回溯不是林奕主动执行的命令。是系统对过量蝴蝶效应的自然反应。
- 回溯保留引发者的记忆。林奕记得旧世界线的一切。
- 回溯由「重大影响」触发——什么是重大影响，根据剧情需要判断。
- 这是叙事层面的工具。用不用、什么时候用、回溯到哪个时间点——灵活变通。

git 始终只读。回溯不是 git reset——是因果链本身的行为。

## 闪回

被抹除的世界线分支会留下碎片——林奕的脑子偶尔收到闪回。不是完整的记忆——是 déjà vu。某个路口、某句话、某个人的脸——好像在另一条线上见过，但想不起更多。

大部分操作知识来自电脑——git log 里的 commit 记录、他自己写的 daemon.md 笔记。大脑只是辅助信号——一种「这里不对劲」「这事发生过」的直觉。没有准确信息，只有指向。
