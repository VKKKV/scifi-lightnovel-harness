# 第16章 · kiri

CTF 之后林奕和叶岚没再说过话。

比赛聊天系统里的对话记录停在「这次没写错。」他没有再找她。她也没有再找他。两个人在同一个赛场上见过，在同一个聊天窗口里说过话——然后各回各的终端前。像是两个在走廊里擦肩的人，点了一下头，然后继续走各自的路。

林奕不确定这算不算认识。

但他一直在想高柳说的话。「先搞清楚 git log 能告诉你什么。」

周三之后他一直在想这句话。不是渗透。不是规则。是他从未认真看过的东西——git log 本身。

但他把 `yan.` 这个 ID 记在了 daemon.md 的末尾。不是规则。是一行注释。

***

比赛后第三天，林奕在做一件他做过很多次但从来没仔细看的事——翻 git log。

不是查某条具体的 commit。是从头到尾看一遍。从第一天到最新一条。

他之前只看最新的几条、和自己操作相关的几条。这次不一样——高柳说「先搞清楚 git log 能告诉你什么」。他开始真的看。

第一条 commit——他第一次渗透的自动记录。时间戳不是正常日期格式——是一个相对值。`t+0.000`。后面跟着他看不懂的几层嵌套编码。commit message 是 hex。解码之后是 `init`。

往下翻。第二条。第三条。他的每次渗透，每次 `observed` 标记，每次系统自动记录的流量异常——全在。

翻到中间的时候他停了下来。

有一条 commit 不对。

不是 hash 的格式不对——hash 和他的 commit 一样长。不是时间戳不对——时间戳在他自己的某条 commit 之后几分钟。是 author 不对。

他之前从没看过 author。

每条 commit 的 author 栏他默认是空的——或者是他自己的 key。他从来没确认过。但这条不一样。author 栏里写着一个名字。

`kiri`

不是他的操作。不是系统自动标记。是另一个人的 commit——出现在他的 git log 里。

林奕把 P53 的屏幕往自己这边拉近了一点。

他往前翻。找到第一条 `kiri` commit。时间戳——2018 年 3 月。

比他拿到 P53 早了五年。

再往前翻。2017 年。2016 年。最老的一条——2013 年 11 月。

kiri 在他之前就已经在用这个系统了。

比 Ryou 还早。

他筛出所有 kiri 的 commit。用 `git log --author=kiri`——命令跑通了。输出了一大堆。分布在整个世界线树上——不是某一条分支，是所有分支都有。commit message 全是数字：1、2、3、4……一直到 47。

他不知道这些数字意味着什么。操作编号？事件编号？还是某个实验的阶段编号？

他在 daemon.md 里加了一行。

`git log 里有不属于我的 commit。作者是 kiri。不知道是什么。分布在整个世界线树上。有些比 2019 年更早。`

敲完他盯着那行字看了一会儿。

kiri。四十七次操作。跨越至少六年。比他早。可能比 Ryou 也早。

他不知道 kiri 是谁。不知道 kiri 现在在哪。不知道 kiri 的 commit 意味着什么。

但他知道一件事——他不是第一个。
