# 第4章 · ssh

林奕不想看第三遍。

如果千夏再死一次——那就是他做的。

而改写的人是他。

凌晨敲的那行命令。他必须弄清楚。

他坐在 P53 前。窗外的路灯已经完全亮了。赵逸风吃完泡面回房间了——门底下的光灭了。公寓里只剩下 P53 屏幕的光和林奕键盘的声音。

他先翻了 git log。

git 命令在 P53 上一直能用——不是他装的，是原主人留下的。他之前没在意——以为只是普通的本地仓库。但昨晚他发现不对。这个 git 仓库没有 `.git` 目录。没有 remote。但它有 log——一条一条的 commit 记录，时间戳不是他认识的任何时区，commit message 不是英语也不是中文是一串十六进制。

他试过 `git show`。每条 commit 展开之后是网络流量——IP 包、TCP 握手、HTTP 请求。像是有人在用 git 记录网络事件。不是他。是这台机器自己记的。

他翻到最近的记录。最新的 commit 时间是几分钟前——千夏第二次被撞的时间。但 commit 的内容不是车祸数据——是他凌晨敲的那行命令。像是 git log 把「操作」和「结果」记在了一起：操作时间是凌晨，事件时间是现在。两个时间戳。一条 commit。

他盯着屏幕。

那行命令不是发出去的 ping。是被这个系统——这台 P53，wl0——接收并执行了。不是发给目标的攻击指令。是发给系统的操作指令。他凌晨以为自己在「试探」——实际上他操作了这个东西。像一个从来没碰过车的人坐进了驾驶室，随便动了一个拨杆。

车撞了。

他现在要做的是倒回去。不是删掉那条 commit——git 删不掉，他试过。是找到那个路口。找到那辆卡车。在时间到达下午四点十二分之前——在那个路口之前——做点什么。

他把 git log 往回翻。翻到第一次千夏死的那天。同一个 commit。同一个 HTTP POST。同一个时间点。

再往前翻——什么都没有。系统记录里只有他凌晨敲的那条命令和它引发的后果。

也就是说，如果他什么都不做——今天下午四点十二分，千夏还是会死。如果他做了——他不知道会发生什么。

林奕做了选择。

他开始分析那条 commit 里的网络流量。目标 IP 不是公网地址——`10.13.x.x`，是一个内网。但 IP 包里有时间戳——不是标准 NTP 格式，是某种自订的编码。他对照着现实时间推了一下：那个 IP 对应的不只是一个网络地址。它对应的是「今天下午四点十二分，千代田区神保町二丁目路口」的整个网络环境。监控摄像头。交通信号控制器。所有的东西。

这个 IP 地址不是位置。是时间。

他敲了 `ssh root@10.13.x.x` 通过 wl0 接口发出去。

以前不会响应的。这次有反应了。

不是欢迎信息。不是 MOTD。是 `Connected`。就这一个词。

林奕的手指停在键盘上。他看着那行字——`Connected to 10.13.x.x`。没有远程 shell。没有端口转发。终端上开始刷数据包了。wl0 接口有流量。ARP 请求。TCP SYN。他在一个网络里。一个物理上不存在于此刻的网络里。时间往前提了六个小时。他在下午四点十二分的路口。

他开始扫描。`nmap` 跑得比正常快得多。每扫一个端口，结果立刻返回。监控摄像头：Hikvision，端口 80——Web 管理界面，端口 554——RTSP 视频流。交通信号控制器：Modicon M340，端口 502——Modbus 协议。路口的路由器：Cisco，端口 23——Telnet。

他先进了监控摄像头。Hikvision 的 Web 界面——默认密码没改。admin:12345。进去了。实时画面：路口正常。千夏不在画面里。右下角时间戳：15:58:42。距离卡车出现还有十四分钟。

他翻到录像回放。找到了上一次——不对，上两次——的录像。第一遍：千夏从杂货店出来，走到路口，站在人行道边等红灯。四点十二分零三秒——卡车闯红灯。她没来得及反应。第二遍：完全一样。连她站的位置都一样——人行道边缘那个有点松的地砖旁边。她两次都踩到那块砖，然后脚尖轻轻压一下，好像在想什么事。然后抬起头。然后卡车。

林奕把录像关了。回到摄像头的系统设置页面。

系统时间。他可以改。Hikvision 的 Web 后台没有权限隔离——能登录就能改系统时间。他把摄像头的时钟往回拨了三秒。15:58:42 → 15:58:39。

他不知道改了之后会发生什么。摄像头的时间戳和交通信号控制器用的是同一个 NTP 源——路口网络里有一台 Cisco 路由器做时间同步。他改的不是信号灯。是信号灯的时钟参考源。如果控制器的周期检查发现时间偏差超过阈值——它会进入校准模式。校准期间，信号周期会重置。红灯的起始点会偏移。

他不知道偏移多少。但他算过：如果红灯晚开始三秒——卡车到达路口的时候灯还是绿的。它会直接开过去。千夏会在绿灯亮之后才走到人行道边——但卡车已经过去了。

他点了保存。

终端上没有反馈。

但 `git log` 里多了一条 commit。自动的。没有 commit message——只有时间戳。时间是今天下午四点十一分五十九秒。

林奕盯着那行新的 commit。然后关掉终端。拿起外套。出门。

***

杂货店的门铃响了。叮——。

千夏在柜台后面。白色长袖 T 恤。头发扎起来。围裙上别着名牌：浅羽。她在整理货架——把日期快到前面的往前摆。动作不快。仔细。

「欢迎光——」

林奕站在门口没动。他看着她在柜台后面转身。看着他。表情是那种「你来了啊」的正常——不是感激。不是劫后余生。是正常的。因为她不知道自己死过。

「金枪鱼饭团——」她指了指货架，「今天到了新的。要试试吗。」

林奕看着她。看了两秒。然后走过去拿了饭团和绿茶。放在柜台上。

她扫条码。滴。饭团。滴。绿茶。找零。塑料袋。

「谢谢惠顾。」

「……谢谢。」

他拎着袋子走到门口。回头。

她在柜台后面翻本子。左手按着纸，右手握着圆珠笔。写了两个字。抬起头看了一眼墙上的门。继续写。

UPS 绿灯闪。

林奕推门出去。

街上没有卡车。贩卖机还立着。罐装咖啡在机器里排得整整齐齐。人行道上没有保温毯。没有血。空气里有十月底的凉意和杂货店门口烤红薯机飘出来的甜味。很淡。

林奕在门口站了几秒。风吹过来——十月底的凉意还在。他把饭团袋子换到另一只手。走了。

***

回公寓之后他打开 P53。

`git log` 多了一条 commit。不是他手动提交的。是系统自动记录的。commit 的内容是他刚才的操作——摄像头 IP、修改的系统时间、时间戳。commit message 栏里有一行十六进制。解码之后是一个数字：1。

第一条。

他在 `~/notes/daemon.md` 里敲了一行。

`第一条：改了一个人的死亡。不是删数据。不是测试边界。是改了一个人的死亡。`

敲完他停了一下。光标在那一行末尾闪。

他没有写「我救了一个人」。他写的是「改了一个人的死亡」。因为他不知道改了之后会有什么后果。git log 上多了一个 commit——但 commit 是可以 revert 的。

他只知道一件事：那行凌晨敲的命令不是试探。是操作。而操作没有撤销键。

P53 的风扇转了一下。停了。赵逸风房间的鼾声也停了——翻了个身，弹簧响了一下，又继续了。

林奕把 daemon.md 关了。打开一个新的笔记。

标题：规则。

第一条：每次操作会被记录。记录就是操作本身——没有撤销。

第二条：待确认。
