# 第21章 · 七次

林奕又做了一次渗透。

很小的操作——改了一个论坛用户的注册时间。不是他认识的人。不是他认识的时间点。他只是想测试：在知道 kiri 存在之后，系统的行为有没有变化。

git log 自动 commit 了。和之前一样。

他翻了翻刚刷出来的 log。忽然看到一个之前没见过的字段——在 commit message 的结尾。回车之前。一行小字。

`death_node`

他没有见过这个标记。

他以为是渗透产生的。往回翻——找到了第一条带 death_node 的 commit。不是他刚才的操作。是他第一次救千夏的那一条。时间戳：t+0.000。

他盯着那行字。

然后他做了一件之前没做过的事——筛出所有带 death_node 的 commit。`git log --grep=death_node`。

输出了一堆。

不是一个分支。不是一个时间点。是多个——不同的分支，同一个名字反复出现。

浅羽千夏。

他一条一条地看。

第一条。监控时间戳修改后的分支。千夏活着——到某个时间点后中断。

第二条。删帖操作后的分支。千夏活着——另一条中断。

第三条。没有他干预的分支。千夏死在那辆卡车下。

第四条。另一条没有他干预的分支。千夏死在另一天。另一个位置。

第五条。不同的死法。

第六条。

第七条。

七种方式。七条分支。七次死亡。

在每一条他没有干涉过的世界线上——千夏是必死的。

不是意外。不是概率。是节点。她在每个因果链的某个位置都会被终止——不管他怎么改。不同的方式。同一个结果。

林奕看着屏幕。

他的手没有抖。他把 daemon.md 打开了。翻了翻里面关于千夏的记录——和她相关的渗透记录、监控结果、每天在便利店确认她活着的笔记。他按了两下快捷键。全部选中。删除。daemon.md 只剩下最后一行。他敲的。

`我不知道为什么。但我不会再让她死。`

他合上 P53。

窗外天快亮了。赵逸风的闹钟还有一个小时才响。公寓里安静得像一个没有活人的房间。

林奕坐在黑暗里。

七次。

他不知道为什么。但他知道自己接下来要做什么。
