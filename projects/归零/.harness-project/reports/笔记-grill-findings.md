### 模式总结（通用）

#### 高柳类学术角色写作原则

高柳（东工大量子信息科学教授）的知识边界是外部研究者的典型案例：

**可以知道：**
- 理论/数学框架（退相干历史、世界线分支理论）
- 可观测数据模式（commit 频率、anomaly 标记分布）
- 从公开信息推导（论坛帖子内容、MAC 地址中的日期）
- 假设和推测（「如果真实存在，应该表现为 X」）

**不能知道：**
- 系统内部设计原理（自适应接口、语言模型适配）
- 系统设计哲学（「系统为了方便你理解……」）
- 只有组织或建造者才知道的实现细节
- kagure 的私信内容或其他角色的非公开信息

**检验方法（两问）：**
1. 这个信息是通过学术方法（理论推导/数据观测/公开资料）能得出的吗？
   - 能 → 保留。不能 → 删除。
2. 这个信息如果出现在学术论文里，专业人员会惊讶吗？
   - 不会 → 保留。会 → 需要来源或删除。

**正确写法示例（来自 Ch13 修正版）：**
```
「没有。」高柳把咖啡罐放下。「我没有见过那个接口。但我推导过它存在的依据。」
```
→ 不声称知道是什么，只推导存在的依据。后面只问数据、讲理论框架、给警告（「别试 git branch」），不解释内部机制。
