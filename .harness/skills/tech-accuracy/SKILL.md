# 科幻技术自洽审查

> 审查黑客/计算机技术描写的准确性。在设定的抽象层级上，命令/工具/攻击链是否真实可信。

## 检查维度

### 1. 术语准确性
- □ CTF 术语是否正确使用（SQLi/Pwn/Rev/Web/Crypto/Misc）？→ ★★ 严重
- □ Unix 命令是否真实存在且用法正确？→ ★★ 严重
- □ 是否编造了不存在的工具/命令（可虚构未来工具但需合理）？→ ★ 建议

### 2. 渗透流程合理性
- □ 攻击链是否合理（信息收集→漏洞利用→提权→覆盖痕迹）？→ ★★ 严重
- □ 渗透难度是否与角色当前技术水平匹配？→ ★★ 严重
- □ 是否出现「一键 hack」式不合理的简化？→ ★★ 严重

### 3. 终端界面一致性
- □ wl0/IP 地址格式是否与设定一致（数字编码时间偏移）？→ ★★ 严重
- □ git log 输出格式是否合理（commit hash/date/message）？→ ★ 建议
- □ daemon.md 的笔记风格是否与林奕的技术水平一致？→ ★ 建议

### 4. 技术成长曲线
- □ 林奕的技术进阶是否有学习过程（不能跳级）？→ ★★ 严重
- □ kagure 的教学内容是否与他声称的技术水平一致？→ ★ 建议
- □ CTF 题目难度是否真实可考？→ ★ 建议

## 违规等级

| 等级 | 标记 | 含义 |
|:-----|:-----|:-----|
| 严重 | ★★ | 技术错误——影响可信度 |
| 建议 | ★ | 优化建议——提升真实感 |
