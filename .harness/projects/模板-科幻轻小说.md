# 模板：科幻轻小说

> 本文件为科幻轻小说题材的**通用创作约束**。所有该题材的项目自动继承此约束。
> 项目专属约束见 `projects/{项目}/constraints.md`（R-G 系列）。

---

## 题材特征

科幻 + 轻小说 = 硬设定 + 软叙事

- **科幻**：世界线系统、跨时空渗透、蝴蝶效应 → 设定有逻辑，可推导
- **轻小说**：日常感、角色魅力、对话驱动、情感克制 → 叙事有温度

---

## 通用硬约束（该题材下所有项目共享）

### R1 · 科学设定优先用人话（硬）
设定本身的因果应该能用日常语言说清楚——「藏了知识」「改了分支」「她死了」。
量子术语只在角色需要的时候出现（教授的理论解释），不是设定的基础构件。
违反 → ★★★ 致命。

### R2 · 代价 = 蝴蝶效应（硬）
每次渗透 = 蝴蝶翅膀。代价不是系统惩罚，是因果律重算。
波及范围与渗透规模成正比。波及的人 = 主角周围的人。
违反 → ★★★ 致命。

### R3 · 主角靠自己成长（硬）
没有系统给能力。渗透技术 = CTF 实战 + 自己摸索 + kagure 教学。
技术成长有过程，不允许跳级。
违反 → ★★ 严重。

### R4 · 闪回 = 辅佐信号（硬）
大脑中的闪回 = déjà vu，不是完整记忆。大部分操作知识在电脑上。
违反 → ★★ 严重。

### R5 · 日常有功能（软）
每一个日常段落至少推进 1 件事：角色关系 / 伏笔 / 世界观 / 节奏。
不允许纯灌水段落。日常主线比例 ~5:5。
违反 → ★ 建议。

### R6 · 对话 > 叙述（软）
轻小说叙事以对话和场景推进为主。超过 500 字的连续纯叙述块需要检查必要性。
违反 → ★ 建议。

### R7 · 信息释放有节奏（硬）
从角色的理解水平出发——读者和林奕同时知道的事，不让读者提前知道。
卷一：试错推导。卷二：理论框架。卷三：揭示真相。卷四：终极理解。
违反 → ★★★ 致命。

### R8 · 情感用控制写（硬）
最大的情感 = 最小的动作。不允许煽情式表达——无流泪/瘫坐/大段独白/echo 式结局。
违反 → ★★★ 致命。

## 角色设计原则

- 每个角色有 Want / Need / Flaw
- 每个角色有独立于主角的弧线
- 功能重叠的角色 → 合并或降为背景
- 角色反应符合性格——六年独行的 hacker 不会在人前失态

## 叙事结构

- 第三人称限知视角（主角为主，可切关键配角）
- 每章 3000-4000 字
- 卷级结构（~100 章/卷，4 卷）
- 每章结尾有钩子（信息打断/冲突升级/悬念）
